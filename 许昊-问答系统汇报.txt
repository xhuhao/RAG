二、评估方法与结果
使用RAGas进行评估
评估流程：
测试问题 → RAG系统回答 → RAGas评估 → 输出四项指标分数
评估指标和分数：
Faithfulness （忠实度：回答是否基于文档）分数51.51%
Answer Relevancy （回答相关性：回答与问题相关性）分数86..27%
Context Precision (上下文精确度：检索文档精确度) 分数78.33%
Context Recall （上下文召回率：检索文档召回率） 分数52.50%
综合评分：67.50%



二、算法优化（调参）：
1. LLM提示词
(1)基本概念:LLM提示词全称System Prompt，是发给大语言模型的系统级指令；
(2)影响包括：Faithfulness（忠实度：回答是否基于检索到的文档内容）、幻觉程度（LLM生成文档中没有的信息）、Answer Relevancy（回答相关性：回答与问题的相关程度）
(3)优化方向：使用约束提示词包括但不限于
  1. 如果参考文档中有相关信息，请基于文档内容回答
  2. 如果参考文档中没有相关信息，请直接回复"抱歉，未找到相关信息"
  3. 回答要准确、简洁、有条理
  4. 适当引用参考文档中的内容
还有待思考如何继续优化



                                                                                                                                                                     
  2. Temperature

  基本概念：Temperature是LLM生成文本时的"随机性"控制参数，取值范围0-1；
  优化方向：降低Temperature值到合适的最优值，Temperature越低，LLM越倾向于选择高概率词（文档中的词），减少非文档自我生成内容，从而降低幻觉，提高忠实度
  


  3. Top K
  基本概念：Top K是向量检索时返回的最相似文档块数量；
  优化方向：继续调整Top_K值，Top K越大，检索到的文档越多，但可能会包含不相关文档，需要不断尝试平衡出最优值。



  
  4. Chunk Size
  基本概念：Chunk Size是将长文档切分成小块时，每个块的最大字符数；
  优化方向：Chunk Size越小，chunk（片段）数量越多,相关信息更容易被找到；但chunk太小可能切断语义（上下文连在一起才能理解的片段被切断），需配合Chunk Overlap使用。还需不断优化平衡





  5. Chunk Overlap
 基本概念：Chunk Overlap是相邻块之间重叠的字符数；
 优化方向：Chunk Overlap越大，块之间的重叠越多，跨块信息更完整，避免重要信息被切断；但重叠过大会增加存储空间。
